    1. 软件下载

    官网下载地址：

    初学者应该是安装community社区版就行了。

    官网下载的最新安装包不需要java环境，已经内置了。

    不建议汉化，自己下个百度翻译PC版，截图翻译，慢慢克服英语困难。

    2. 学习文档

    我喜欢看官方的文档进行学习，getting-started是入门的好文档。

    还有YouTube视频教程，可以使用字幕翻译功能，英语基础稍微好一点的都能够理解。

    3. 软件显示

    存在高清屏幕导致的字体模糊、字体过小、ui强迫症等因素，第一次使用burp先解决显示问题。

    • 高清屏高DPI配置：桌面右键快捷方式 - 属性 - 兼容性 - 更改高DPI设置 - 高DPI缩放替代 ：应用程序。

    • 拦截器和消息编辑器的字体：右上角settings – user interface – inspector and message editor – HTTP message display – Font ：consolas 11pt

    • 用户界面（菜单栏）的字体：右上角settings – user interface – display – appearance – Font size ：11

    • 保存配置：右上角三点 - user setting - save ，保存json格式的配置。

    • 默认使用json配置：关闭软件再次打开，选临时项目，在加载项目时选择你想要的配置，点击 load from configuration file ，选择你刚才保存的json文件， 再勾选 default to the above in future。

    4. 代理配置

    使用Burp Suite，最重要的是能够捕获到HTTP流量，有两种方法：使用内置浏览器、第三方浏览器代理。

    4.1. 内置浏览器

    Proxy – Intercept – open browser。burp浏览器已预配置代理。

    4.2. 第三方浏览器

    如谷歌浏览器。

    • 打开Burp Suite，右上角Settings – Tools – Proxy – proxy listeners，可以看到端口：8080。

    • 安装谷歌浏览器插件：Proxy SwitchyOmega。

    • 新建情景模式。

    • 安装burp证书：浏览器输入http://burp
    • 右上角下载证书 – 打开安全证书 – 安装证书 – 选择受信任的根证书颁发机构。

    • 浏览器现在可使用Burp Suite进行代理上网。

    • 打开Burp Suite，查看第二行的Proxy – HTTP history，可以看到代理的流量。

    • 上游代理：setting - network - connections - Upstream proxy servers，配置翻墙软件的端口，让burp作为客户端再进行代理一次。

    5. 功能介绍

    5.1. Proxy

    首先是代理，这里是核心的功能，可以使用intercept拦截器，拦截本机和目标服务器的HTTP流量，通过Forward、Drop、Action进行对HTTP包的研究分析。

    • Intercept

      默认Intercept is off，开启拦截器Intercept is on，拦截访问目标server的流量。（如果你发现网络突然中断了，那应该就是Intercept is on。）

      使用forward转发流量到目标服务器，使用drop丢弃该请求包，使用action进行进一步研究分析，action如下图。

    • HTTP history

      HTTP历史记录显示了浏览器发出的每个请求的详细信息。

      点击左边的#，切换倒序，即可实时查看最新的流量。

      可以通过Filter settings进行过滤；可以右键highlight，选择某一条标记颜色，方便寻找。

      配置scope，在过滤配置里勾选only in-scope，可以使HTTP history更精简。

    • WebSockets history

      略。

    • Proxy setting

      配置正则表达式，match and replace rules，自动更换字符。

      配置拦截返回的信息：response interception rules - intercept responses based on the following rules。

    • Inspector

      这个是一个便捷的包分析器，你可以根据HTTP的参数来修改对应的value。这与直接在request、response包里修改参数的值是一样的。

    5.2. Target

    在Target – Site map ，将HTTP流量归纳到了不同的站点（域名）。

    • 选择某站点右键add to scope。选择目标站点，作为本次分析工作的感兴趣URL范围。
    • 出现弹窗选择no。继续记录其他网页的访问流量。
    • 在scope setting – target scope，可以看到添加的URL。
    • 在HTTP history – Filter settings，勾选show only in-scope items，则仅显示感兴趣流量，减少多余的繁杂信息。

    5.3. Repeater

    Burp Repeater 是一个工具，使您能够反复修改和发送有趣的 HTTP 或 WebSocket 消息。（应保证代理正常，这样才能抓取回包。）

    • 在HTTP history选择特定的HTTP包。
    • 右键add to repeater。
    • 在Repeater标签，修改某一参数，send。
    • 在send按钮右边，可以选择历史尝试的内容。

    5.4. Intruder

    Burp Intruder 是一种针对 Web 应用程序自动执行定制攻击的工具。它使您能够配置多次发送相同 HTTP 请求的攻击，每次将不同的有效负载插入到预定义的位置。

    • 在HTTP history选择特定的HTTP包。

    • 右键add to intrude。

    • 设置Positions。

      主要界面，在右侧可以选择一个或多个参数进行攻击。

      可以选择攻击类型，根据payload，进行规则或不规则的攻击顺序。暂不展开，内容较多。

    • 设置Payloads。设置攻击的参数值，可设置一组或多组的值。

    5.5. Dashboard

    配置扫描任务。

    主动扫描（pro版）、被动扫描。
